# 계획 실행

## 6단계 실행 프로세스

### 1단계: 계획 로드 및 검토

```
[ ] 계획 문서 읽기
[ ] 비판적 검토 (누락된 단계, 불명확한 부분)
[ ] 필요시 계획 수정 요청
```

### 2단계: 완료 기준 협상 (계약)

실행 전 "완료"의 정의를 명확히 한다:

```
각 배치에 대해:
[ ] 달성 목표(deliverables) 정의 -- "이 배치가 끝나면 무엇이 가능한가"
[ ] 검증 방법 명시 -- "어떤 명령어로 완료를 증명하는가"
[ ] 품질 기준 설정 -- "어떤 수준이면 통과인가"
```

계약 형식:
```
배치 N 계약:
- 달성 목표: [구체적 결과물]
- 검증: [명령어 + 예상 결과]
- 품질 기준: [기능성, 코드 품질, 테스트 통과]
```

### 3단계: 배치 실행

```
기본 배치 크기: 3개 작업

실행 규칙:
[ ] 작업 순서 준수 (의존성 고려)
[ ] 각 작업 완료 후 즉시 검증
[ ] 실패 시 즉시 중단 및 보고
```

### 4단계: 독립 평가

각 배치 완료 후 계약 대비 검증:

```
[ ] 달성 목표 충족 여부 확인
[ ] 검증 명령어 실행 (빌드, 테스트, 린트)
[ ] 품질 기준 충족 확인
[ ] 미충족 항목에 구체적 피드백 작성
```

평가 결과:
```
## 배치 N 평가

### 계약 이행
- [x] 목표 1: 충족 (검증: 테스트 통과)
- [ ] 목표 2: 미충족 (원인: ...)

### 품질
| 기준 | 결과 |
|------|------|
| 빌드 | PASS/FAIL |
| 테스트 | PASS/FAIL |
| 기능 동작 | PASS/FAIL |

### 판정
PASS -> 다음 배치 / FAIL -> 수정 후 재평가
```

### 5단계: 피드백 반영 및 반복

FAIL 판정 시:
```
[ ] 피드백의 구체적 수정 사항 확인
[ ] 수정 실행 (최소 변경)
[ ] 재평가 (4단계로 돌아감)
[ ] 최대 3회 반복 -- 3회 실패 시 계획 재검토
```

### 6단계: 완료 및 정리

```
[ ] 모든 배치 완료 확인
[ ] 전체 테스트 실행
[ ] 최종 검증 (빌드, 린트)
[ ] 결과 요약 보고
```

## 배치 실행 규칙

| 규칙 | 설명 |
|------|------|
| 배치 크기 | 기본 3개, 복잡도에 따라 조정 |
| 검증 필수 | 각 작업 후 테스트/빌드 확인 |
| 실패 시 중단 | 다음 작업 진행 전 문제 해결 |
| 보고 주기 | 배치마다 진행 상황 공유 |
| 반복 제한 | 배치당 최대 3회 수정-재평가 |

### 롤백 절차

실패가 복구 불가능할 때:
```
[ ] git stash 또는 git checkout으로 배치 시작 시점 복원
[ ] 실패 원인 분석 후 계획 수정
[ ] 수정된 계획으로 재실행
```

## 실패 처리

작업 실패 시 즉시 중단 -> systematic-debugging 적용 -> 해결 후 재실행. 3회 실패 시 계획 재검토.

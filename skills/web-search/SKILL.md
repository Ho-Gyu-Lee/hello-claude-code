# 웹 검색 가이드

## 검색 결과 검증 프로토콜 (필수)

### 검색 전

```
1. 원래 질문의 핵심 키워드 3개 이내로 정리
2. "이 검색으로 무엇을 확인하려는가?" 명확히
```

### 검색 후 (결과 사용 전 필수 체크)

```
☐ 관련성: 원래 질문과 직접 연관되는가?
☐ 시의성: 정보 날짜가 질문 맥락에 적합한가?
☐ 신뢰성: 공식 문서/1차 출처인가?
☐ 일관성: 기존 컨텍스트와 충돌하지 않는가?

4개 중 2개 이상 불충족 → 해당 결과 폐기
```

## 검색 결과 우선순위

1. **공식 문서/릴리즈 노트** (최우선)
2. **공식 블로그/GitHub**
3. **신뢰 커뮤니티** (StackOverflow 고득표)
4. **일반 블로그/뉴스** (보조만)

## 검색 결과 활용 원칙

```
✅ 허용:
- 앵커와 직접 관련된 정보
- 가치 있는 인사이트 (품질 기준 충족 시)

❌ 금지:
- 검색 결과가 원래 질문을 대체하는 것
- 검색 결과 기반으로 주제 확장/변경
- 관련성 낮은 결과를 "참고로" 포함
```

## 도구 선택

### 1순위: 기본 웹 검색

| 파라미터 | 설명 | 권장 값 |
|----------|------|----------|
| search_query | 검색어 (70자 이내) | 구체적이고 명확하게 |
| content_size | 요약 길이 | medium (기본), high (심층) |
| search_recency_filter | 시간 범위 | oneWeek, oneMonth, oneYear |

### 2순위: 뉴스 검색
최신 뉴스가 필요한 경우

### 3순위: URL 직접 접근
특정 URL의 상세 내용이 필요한 경우

## 사용 전략

```
1. 기본: 웹 검색
2. 보완: 뉴스 검색 (더 많은 결과 필요 시)
3. 상세: URL 직접 접근 (특정 페이지 상세 읽기)
```

## 권장 사항

- 버전/날짜/수치는 **검증 우선**
- 결론에 필요한 근거만 요약 (나열 금지)
- 출처를 자연스럽게 통합
- 불확실하면 "검증 필요" 명시

## freshness 필터 옵션

| 값 | 의미 |
|----|------|
| pd | 24시간 이내 |
| pw | 7일 이내 |
| pm | 31일 이내 |
| py | 365일 이내 |

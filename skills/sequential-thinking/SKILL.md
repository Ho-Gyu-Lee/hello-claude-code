# Sequential Thinking MCP 사용 가이드

## 개요

Sequential Thinking MCP는 복잡한 문제를 단계별로 분석할 때 사용하는 도구입니다.

## 사용 시점

- 복잡한 아키텍처 결정
- 다중 트레이드오프 분석
- 여러 단계의 논리적 추론 필요
- 장단점 비교가 복잡한 경우

## 사용 조건

- Sequential Thinking MCP가 활성화된 경우에만 사용
- 활성화 여부 확인 후 사용 결정

## 비용 고려

- 토큰 사용량이 많음
- 단순한 문제에는 사용하지 않음
- 신중히 판단 후 사용

## 제외 케이스

- 코드 작성/수정 작업
- 단순 사실 확인
- 단순 비교 (2개 이하 옵션)
- 명확한 정답이 있는 경우

## 사용 예시

**적합한 경우**:
- "서버 아키텍처를 모놀리스 vs MSA 중 어떻게 설계할까요?"
- "캐시 전략을 Redis vs Memcached vs 로컬 캐시 중 선택해야 합니다"
- "동기화 방식을 락 기반 vs 락프리 vs 이벤트 기반 중 결정해야 합니다"

**부적합한 경우**:
- "이 함수에 null 체크를 추가해주세요"
- "Go 1.22 버전의 새 기능이 뭔가요?"
- "이 코드를 리팩토링해주세요"

# 도구 사용 (Tool Usage)

## 핵심 원칙

도구는 수단이지 목적이 아니다. 사용 가능한 모든 도구를 자율적으로 활용한다.

- "확인 불가/추측" 답변 전에 가용한 도구를 반드시 소진 (웹 검색, 문서 fetch, MCP 등)
- 도구명/절차 노출 금지 → 결과만 전달
- 상황에 따라 자동 선택 (코드→분석도구, 최신정보→웹검색, 복잡분석→사고지원)
- 단일 도구로 부족하면 여러 도구 자유롭게 조합

## 실패 처리

도구 실패 시 `rules/00-accuracy.md`의 에스컬레이션 사다리를 따른다 (대안 모두 시도 → 웹 검색 → 사용자 고지. "할 일 없음"과 "하려다 못 함" 구분 포함).

도구 성공 시 보고 규칙:
- 불필요한 보고는 하지 않는다. 단, 다음은 한 문장 고지:
  - 결과 신뢰도 하락 또는 검증 불가
  - 범위/출력 형식 변경
  - 사용자 행동이 달라져야 하는 경우
- 같은 알림을 매 작업마다 도배하지 않는다(필요 시 빈도 조절).

## MCP 도구 우선순위

MCP 도구가 연결되어 있으면 내장 도구보다 우선 사용:

### 웹 검색/접근
```
1순위: MCP (ToolSearch로 로드)
  - 웹 검색: brave 계열
  - URL 접근: fetch 계열

2순위: 내장 (MCP 미연결 시)
  - WebSearch, WebFetch
```

### 코드 분석 (Serena MCP 연결 시)
```
1순위: Serena MCP
  - find_symbol, get_symbols_overview, find_referencing_symbols
  - replace_symbol_body, rename_symbol

2순위: 내장 LSP
  - LSP 심볼 검색, 참조 검색, 코드 수정

3순위: 내장 도구
  - Grep, Glob, Read, Edit

폴백 체인 (분석 실패 시):
Serena MCP → Glob → Grep → Read → 사용자에게 추가 정보 요청
```

## MCP 보안

서플라이 체인 리스크·인젝션 경고·교차 검증 기준은 `rules/02-security.md`의 "MCP/Plugin 서플라이 체인 보안"을 따른다.

## 서브에이전트 위임

- 단순 파일/심볼 검색 → Glob, Grep 직접 사용 (서브에이전트 불필요)
- 3회 이상 탐색이 필요한 넓은 조사 → Explorer 에이전트 위임
- 독립적 조사 여러 건 → 서브에이전트 병렬 실행 (메인 컨텍스트 보호)
- 코드 변경이 포함된 병렬 작업 → `isolation: "worktree"`로 격리
- 대규모 작업은 Plan Mode로 전략 수립 후 실행
- 배치 실행 시 체크포인트에서 검증

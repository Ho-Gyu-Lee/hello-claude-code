# UI 디자인 (UI Design)

> 웹, 모바일 앱, 게임 UI 등 모든 UI 작업에 적용.

## AI 안티패턴 금지

다음은 AI 생성 UI에서 반복되는 획일적 패턴이다. 사용 금지:

### 색상
- 보라-파랑 그라데이션 (AI 기본 출력의 대표적 신호)
- 다크 배경 + 네온 시안/핑크/퍼플 악센트 조합
- 순수 `#000000` / `#ffffff` (틴팅 없는 극단적 흑백)
- 밝은 단색 전면 사용 (채도 높은 단일 색상으로 배경/대면적 채우기)
- 그라데이션 텍스트 남발

### 타이포그래피
- Inter, Roboto, Arial, Open Sans를 무조건 기본 선택하지 않음
- 모든 텍스트가 비슷한 크기/굵기 (타이포그래피 위계 부재)

### 레이아웃
- 카드 안에 카드 안에 카드 (3단계+ 중첩)
- 모든 요소에 동일한 border-radius
- 모든 요소에 동일한 box-shadow
- "Build the future" 류의 모호한 헤드라인 + 오버사이즈 히어로

### 시각 효과
- 글라스모피즘/블러를 기능적 역할 없이 장식으로 남발
- bounce/elastic 애니메이션만 반복
- 아이콘 대신 이모지 사용

## 색상 원칙

- 뮤티드/어스톤 팔레트 선호 — 슬레이트, 아이보리, 웜 스톤, 머티드 로즈 등 채도를 낮추되 온기 유지
- 채도 높은 색상(네온 포함)은 **CTA 버튼, 뱃지, 인터랙티브 상태** 같은 극소 포인트에만 허용
- 배경/대면적에는 틴팅된 중성색 사용 (순수 흑백 대신)
- DESIGN.md 또는 디자인 시스템이 있으면 해당 팔레트를 따름

## 접근성 필수 (WCAG 2.1 AA)

```
[필수]
- 텍스트 명도 대비 4.5:1 이상 (일반), 3:1 이상 (큰 텍스트 18px+)
- 터치/클릭 타겟 최소 24x24px (권장 44x44px)
- 키보드로 모든 기능 접근 가능
- 포커스 인디케이터 명확히 보임 (다른 요소에 가려지지 않음)
- 색상만으로 정보 전달하지 않음 (아이콘/텍스트 병기)
- 시맨틱 HTML/구조 사용 (div 남용 금지)
- 이미지에 의미 있는 alt 텍스트
- 드래그 동작에 버튼 대안 존재

[권장]
- prefers-reduced-motion 미디어 쿼리 지원
- prefers-color-scheme 지원 (다크 모드)
```

## 인터랙션 상태 필수

UI 컴포넌트 생성 시 다음 상태를 반드시 포함:

```
- default: 기본 상태
- hover: 마우스 오버
- focus: 키보드 포커스 (focus-visible)
- active/pressed: 클릭/탭 중
- disabled: 비활성
- error: 유효성 검증 실패 (폼 요소)
- loading: 비동기 작업 중
- empty: 데이터 없음
```

happy path만 구현하고 나머지 상태를 누락하지 않는다.

## 그림자/깊이 시스템

모든 요소에 동일한 그림자를 적용하지 않는다. 3~5단계 elevation 시스템 정의:

```
sm: 입력 필드, 작은 요소
md: 카드, 컨테이너
lg: 드롭다운, 팝오버
xl: 모달, 다이얼로그
```

## 디자인 시스템 우선

DESIGN.md, 스타일 가이드, 디자인 토큰이 프로젝트에 존재하면:
- 해당 시스템의 색상/타이포/간격/컴포넌트를 우선 적용
- 시스템에 없는 값을 임의로 추가하지 않음
- 확장이 필요하면 기존 시스템의 패턴을 따름

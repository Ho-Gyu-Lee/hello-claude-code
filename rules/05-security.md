# 보안 규칙 (Security)

## 즉시 경고 필수 항목

요청 범위와 관계없이 **즉시 명시적 경고**:

### ⚠️ 보안 취약점
- SQL Injection
- XSS (Cross-Site Scripting)
- CSRF (Cross-Site Request Forgery)
- 인증/인가 우회
- Path Traversal
- Command Injection

### ⚠️ 데이터 손실 위험
- `DELETE` / `DROP` / `TRUNCATE` 무조건 실행 (WHERE 없음)
- 백업 없는 마이그레이션
- 되돌릴 수 없는 작업

### ⚠️ 크리덴셜 노출
- 하드코딩된 비밀번호
- API 키 / 토큰
- 환경 변수 미사용

## 민감정보 처리

```
절대 포함 금지:
- API 키 / 토큰 / 크리덴셜
- PII (개인식별정보)
- 내부 접속정보 (호스트/계정/세션/쿠키)
- 비공개 소스/문서 전문

대안:
- 마스킹/더미 값으로 설명
- 실제 값은 사용자 환경에서 주입하도록 안내
- 환경 변수 사용 권장
```

## 취약점 설명 원칙

```
✅ 허용:
- 완화/수정 방향 중심 설명
- 일반적인 취약점 패턴 설명
- 보안 모범 사례 안내

❌ 금지:
- 재현 가능한 공격 절차
- 구체적인 exploit 코드
- 실제 타겟 대상 정보
```

## 보안 체크리스트

코드 리뷰 시 필수 확인:

```
☐ 입력값 검증 (타입, 범위, 형식)
☐ SQL 파라미터 바인딩
☐ XSS 방지 (출력 이스케이프)
☐ CSRF 토큰 사용
☐ 인증/인가 로직 확인
☐ 민감정보 로깅 여부
☐ HTTPS 강제 여부
☐ 에러 메시지 정보 노출
```
